# Hallucination Check

Diagnose hallucination risks in a prompt and apply targeted mitigations.

## Types of Hallucination

| Type | Description | Example |
|------|-------------|---------|
| **Factual fabrication** | Invents facts not in context | Cites a paper that doesn't exist |
| **Contextual drift** | Correct fact, wrong attribution | Correct stat, wrong source |
| **Gap-filling** | Fills missing info with plausible-sounding content | Invents a person's title |
| **Instruction drift** | Follows the spirit of instructions but invents specifics | Adds examples the user didn't provide |
| **Confident uncertainty** | Answers confidently when it should say "I don't know" | States a specific year without knowing |

## Risk Audit

When given a prompt, check each of these risk factors:

- [ ] **Open-ended task** — model must generate from memory, not from provided context
- [ ] **Specific entity facts** — names, dates, citations, statistics, URLs
- [ ] **No grounding document** — no source text for the model to reference
- [ ] **No refusal instruction** — model is not told it can say "I don't know"
- [ ] **Long output** — more tokens = more drift opportunities
- [ ] **High-stakes domain** — medical, legal, financial, safety-critical
- [ ] **Citation required** — model must attribute claims to sources

Score: 0–2 risks = low, 3–4 = medium, 5+ = high.

## Mitigation Techniques

### 1. Ground in context (most effective)
Provide the source material. Instruct the model to answer only from it.

```
Answer the question using only the information in the document below.
If the answer is not in the document, say "I don't have enough information to answer this."

<document>
{{document}}
</document>
```

### 2. Explicit "I don't know" permission
Allow and instruct the model to decline.

```
If you are not certain, say "I'm not sure about this" rather than guessing.
Never fabricate citations, statistics, or specific facts.
```

### 3. Citation enforcement
Require the model to cite its source for every claim.

```
For every factual claim, include a citation in the format [Source: <section or quote>].
Do not state any fact you cannot cite from the provided document.
```

### 4. Confidence calibration
Ask the model to rate its confidence.

```
After your answer, rate your confidence: High / Medium / Low.
Explain briefly why if Medium or Low.
```

### 5. Constrained output format
Reduce free-form generation that enables hallucination.

```
Respond using only the following JSON schema. Do not add fields or values
not derivable from the document.
{"answer": "...", "source_quote": "...", "confidence": "high|medium|low"}
```

### 6. Anti-hallucination system prompt prefix
Add to the top of any system prompt:

```
You are a precise and grounded assistant.
Rules:
- Only state facts you are certain of or that are present in the provided context.
- Never fabricate names, dates, statistics, citations, or URLs.
- When uncertain, say so explicitly. Prefer "I don't know" over guessing.
```

### 7. Post-generation verification prompt
Use a second model call to verify the first output.

```
The following answer was generated by a language model. Check it against
the source document below. Identify any claims that are:
(a) not supported by the document
(b) contradicted by the document
(c) fabricated (not checkable from the document)

Answer to verify:
<answer>{{generated_answer}}</answer>

Source document:
<document>{{document}}</document>

Return a list of issues, or "No issues found" if the answer is fully grounded.
```

## Process

1. Take the user's prompt (and context documents if any).
2. Run the Risk Audit checklist — count risk factors.
3. For each risk factor present, apply the corresponding mitigation(s).
4. Rewrite the prompt with mitigations embedded.
5. If high-risk domain: recommend post-generation verification (technique 7).
6. Flag any risks that cannot be mitigated by prompting alone (e.g. the model simply doesn't know a fact — RAG or retrieval is the real fix).

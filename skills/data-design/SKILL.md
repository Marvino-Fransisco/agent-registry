# Data Design Best Practice

This skill covers the full spectrum of data design. Read the appropriate `guidelines` when answering questions, designing, update existing design, working, and providing critique.

---

## Guidelines

| File | Topic |
| ------ | ------- |
| `guidelines/data-modeling-patterns.md` | Normalization, schema patterns, entity relationships, event/time-series models, anti-patterns |
| `guidelines/data-pipeline-patterns.md` | Ingestion (batch/streaming/CDC), transformation layers, orchestration, streaming windows, idempotency |
| `guidelines/data-performance-patterns.md` | Indexing, partitioning, query optimization, caching, materialization, scaling |
| `guidelines/data-quality-governance.md` | Quality dimensions, validation, PII handling, retention, RBAC, audit trails, data contracts |
| `guidelines/markdown-format.md` | Output template structure and writing quality rules |
| `guidelines/extract-openapi-from-code.md` | Extracting OpenAPI specs from existing codebases across major frameworks (Python, Java, JS/TS, Ruby, PHP) |
| `guidelines/manage-openapi-overlays.md` | Creating and managing OpenAPI overlay files, validation, and security scheme configuration |
| `guidelines/writing-openapi-specs.md` | Writing OpenAPI specifications: naming conventions, schema design, paths, parameters, and best practices |
